Operator console

A browseable front end for the live go-ledger API: the ledger views (accounts, transactions, balances, statements) plus the operator surface (tenants, api keys, webhooks, policy) and read-only reporting. Every call here goes straight to the same public /v1 and /v1/admin endpoints anyone can call directly; this page adds no logic of its own.

Public demo: the ledger resets every 4 hours to fresh, realistic sample data. Anything you create here, including tenants and keys, is cleared on the next reset. Admin panels are public in this demo too: there is nothing durable to protect between resets, and the demo key is rate limited.

Session

Stored only in this browser, in local storage under goledger_admin_key. Sent only to this server, as an Authorization: Bearer header on every request. Never sent to any other origin and never logged by this page.

The admin panels below (api keys, webhooks, policy) act on this tenant. Ledger and reporting views always reflect the tenant your active api key itself belongs to, since those endpoints take no tenant parameter of their own; switching tenant here still re-runs any reporting view you have already loaded, and updates the admin panels to the newly selected tenant.

Ledger

Create accounts, post a balanced transaction by picking accounts from a dropdown, and read balances and statements. Scoped to your active key's tenant.

Admin

Manage tenants, api keys, webhook subscriptions, and posting policy. Every route here requires the admin scope; the server is the real gate, this page's panels are shown as soon as a key is present, and any 403 is surfaced plainly rather than hidden.
