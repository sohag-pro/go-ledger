internal/domain/transaction.go 
// postings are signed; they must sum to zero.
func (t Transaction) Validate() error {
    sum := t.Postings[0].Amount
    for _, p := range t.Postings[1:] {
        next, err := sum.Add(p.Amount) // +debit / -credit
        if err != nil {
            return err
        }
        sum = next
    }
    if !sum.IsZero() {
        return ErrUnbalanced // money can't appear or vanish
    }
    return nil
}

// what's inside

Correctness, encoded.

shipped

Double-entry domain model

Invariants encoded directly in Go types: an unbalanced transaction won't compile a path to disk.

shipped

Atomic, concurrency-safe posting

SERIALIZABLE isolation plus row locks, stress-tested under concurrent contention.

shipped

Idempotency keys

An Idempotency-Key makes a retry safe: the same key returns the original transaction and never moves money twice.

shipped

Immutable audit log

Append-only, written in the same transaction as the posting, and enforced immutable by a Postgres trigger.

shipped

REST + gRPC

Two front doors over one shared domain core, so the protocols cannot drift. Same rules, same errors, same guarantees.

shipped

Full observability

One OpenTelemetry trace from HTTP to SQL, the same trace_id on every log line, and RED metrics beside the native ones. No amounts ever leave the process.
